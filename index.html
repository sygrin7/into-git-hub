Git Basics — Quick Reference

This page lists common Git commands with short explanations and examples. Use it as a quick reminder while working in this repository.

Repository & setup

git init

Create a new repository in the current folder.

git clone <url>

Copy a remote repository locally.

Working with changes

git status
git add file.txt
git commit -m "Short descriptive message"

Check status, stage changes, and commit them.

Branches

git branch             # list branches
git branch feature-x   # create new branch
git checkout feature-x # switch to branch
git switch -c feature-x # create and switch (newer syntax)

Syncing with remotes

git fetch origin
git pull origin main   # fetch + merge from remote main
git push origin feature-x

Merging & Rebasing

git checkout main
git merge feature-x
# or
git checkout feature-x
git rebase main

Use merge to combine branches, rebase to replay commits onto another base.

History & inspection

git log --oneline --graph --decorate
git diff               # show unstaged changes
git show HEAD          # show last commit

Undoing changes

git restore file.txt        # discard unstaged changes
git restore --staged file.txt # unstage a file
git reset --hard HEAD~1     # reset to previous commit (destructive)

Stashing

git stash          # save working state
git stash pop      # re-apply saved state

Remotes

git remote -v
git remote add origin <url>

Tips for this repo

  • Run git status before editing to see uncommitted work.
  • Make small, focused commits and use descriptive messages.
  • Open a PR for larger changes and reference this repository's docs (e.g., README.md).
